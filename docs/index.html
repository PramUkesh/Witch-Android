Value diffing

By default, only new values (instances) will be bound to its view. I.e. changing the inner properties of an instance won't qualify it to be bound in upcoming bind pass. A value will only be bound if it is:

  • 1. Not the same instance as currently bound value

  • 2. Not equal to currently bound value

{% highlight java linenos %} {% include snippets/9.java %} {% endhighlight %}

If the value to be bound cannot (or should not) be re-created, it can be wrapped in a Value container. The Value container gives control over when a value should be bound or not. Calling Value.set() or Value.update() will flag the value as dirty and will then be part of next bind pass, regardless of if value is new or different from before.

View holders built in

Witch will create view holders for all annotated views. This eliminates the need for creating additional view holders for adapter views. Simply use the RecyclerViewBinderAdapter that requires no implementation besides the actual data binding.

Since binder will handle binding of all items of type Post in adapter, it's important to separate binder from item. Binder will be updated with a new item before each bind. This happens under the hood and is handled entirely by Witch.
